/ 01 — Theoretical foundation

01. From the perceptron to the universal approximator

A single perceptron computes ŷ = f(w · x + b). With a step activation it’s Rosenblatt’s 1958 form — only linearly separable problems are reachable, and XOR remains its canonical failure mode. Stacking layers h⁽ˡ⁾ = σ(W⁽ˡ⁾ h⁽ˡ⁻¹⁾ + b⁽ˡ⁾) with a non-polynomial activation gives a feed-forward network that is a universal approximator (Hornik et al., 1989).

Backpropagation

Each weight update is the chain rule, applied layer by layer:

∂𝓛/∂Wⁱ = ∂𝓛/∂ŷ · ∂ŷ/∂z⁽ⁱ⁺¹⁾ · ⋯ · ∂h⁽ⁱ⁾/∂z⁽ⁱ⁾ · x   ⟶   w ← w − η · ∂𝓛/∂w

Mini-batch SGD averages the gradient over a small batch — a noisy but cheap estimator that often generalises better than full-batch GD. The platform composites the softmax+CCE and sigmoid+BCE gradients into the numerically stable (ŷ − y) / N form to avoid log/exp blowups.

RBF networks — a different geometry

Radial Basis Function networks place K Gaussian centers cₖ and compute φₖ(x) = exp(−‖x − cₖ‖² / 2σ²). The hidden layer is fixed (centers chosen from data, σ from the median pairwise distance); only a linear output head trains. Fast, easy to reason about, scales poorly with input dimensionality — a clean foil to the MLP.

Bias / variance — the only thing that really matters

Error ≈ Bias² + Variance + Noise

Capacity trades bias for variance; regularisation pushes the other way. The platform highlights both failure modes: underfitting (train and val loss both high) and overfitting (train loss falls while val loss climbs).

/ 04 — Implemented bonuses

04. Beyond the spec

bonus · validation
scikit-learn baselines

Each experiment row trains a closest-equivalent sklearn estimator on the same split. The Experiments tab shows four side-by-side columns: ours · acc, ours · loss, sk · acc, sk · loss.

Validates that the pure-NumPy engine reaches comparable accuracy to a battle-tested implementation, and exposes places where it doesn’t.

bonus · interaction
Live network topology

An animated SVG of the configured architecture; edge thickness/colour reflects weight magnitude/sign and pulses on each snapshot. Students can see learning happen.

Mint = positive, pink = negative, fade ∝ magnitude.

bonus · 3D
Loss landscape · Plotly

Filter-normalised random directions (Li et al., 2018) drawn in flattened weight space — the trained solution sits at α = β = 0; the steepness of the bowl is a proxy for the sharpness of the minimum.

Free extra · RBF network

Beyond the spec: backend/models/rbf.py implements a Gaussian RBF network with centers picked from the training set and σ from the median pairwise distance. It exposes the same model interface, so the shared training loop runs on it unchanged. Pedagogically clean foil to the MLP — fixed hidden layer, only the output head trained.

/ 05 — Critical analysis

05. What works, what doesn’t, what’s next

strengths
What works well
  • Pure NumPy engine — every operation is visible, debuggable, pedagogically transparent.
  • Streaming training — NDJSON lets the UI animate per-epoch progress without polling.
  • Live network visualisation — edges reflect weights and pulse on snapshot.
  • Three mandatory experiments wired in, one click away, with sklearn columns.
limitations
Honest gaps
  • SGD only — no momentum, no Adam. Fine for 2D demos; matters on harder problems.
  • RBF centers are non-trainable — clean for teaching, not state-of-the-art.
  • In-memory model store — sessions vanish on server restart; no persistence layer.
  • No tests, no linter, no CI — explicit in CLAUDE.md.
  • CSV upload is naive — assumes last column is label, drops NaN rows.

Possible extensions

optimiser
Adam alongside SGD

Cheap to implement, opens up an obvious experiment ("SGD vs Adam convergence on circles + spirals").

rbf
k-means + learnable σ

Initialise centers via k-means, optimise σ jointly with the output head — closer to a real RBF.

infra
Persist sessions

Pickle trained MODEL_STORE entries to disk so a demo survives a server restart.

/ 11 — Glossary

11. Terminology

ActivationElement-wise non-linearity applied to z = Wx + b.
BackpropChain-rule walk that produces ∂𝓛/∂Wⁱ for every layer.
DropoutRandomly zeros activations during training to force redundancy.
Early stoppingHalts training when validation loss stops improving (with best-weight rollback).
He initWeight init scaled by √(2 / fan_in) — pairs well with ReLU.
L2 reg.Adds λ Σ w² to the loss — penalises large weights.
Mini-batch SGDGradient descent on small random subsets of the training set.
NDJSONNewline-delimited JSON — one record per line, easy to stream.
RBFRadial Basis Function. Hidden layer = Gaussian kernels at fixed centers.
Universal approximatorA network with one non-polynomial hidden layer can fit any continuous function on a compact set.
/ 12 — Reflections

12. Closing notes — challenges & lessons learned

Neuralab began as a course project and ended up as a full educational studio: a pure-NumPy ML engine, a streaming Flask API, a single-page interface with a live network visualisation, and three one-click experiments validated against scikit-learn. Building it end-to-end forced us to confront neural networks not as a library call but as a sequence of small, debuggable matrix operations — and to design an interface that makes those operations visible while a student watches them happen.

The hardest parts were rarely where we expected. Numerical stability in backprop bit us early: a naive softmax + cross-entropy implementation produced NaNs within a few epochs until we collapsed the composite gradient to the stable (ŷ − y) / N form. Streaming required rethinking how the frontend talks to the backend — moving from request/response to an NDJSON pipe, and sub-sampling weight snapshots to ~30 per run so the wire and the SVG renderer stayed responsive. The filter-normalised loss landscape (Li et al., 2018) was the most subtle: drawing two random directions in flattened weight space and normalising them per parameter so that no single weight matrix dominates the visualisation. And every model — perceptron, MLP, RBF — had to expose the same forward / backward / apply_gradients surface so the optimizer could stay model-agnostic; getting that interface right took more refactoring than we’d planned for.

What we take away: writing the math by hand is the fastest way to actually understand it. Sigmoid derivatives, ReLU’s zero-region pathology, dropout’s inverted-mask trick, He initialisation’s √(2 / fan_in) — these stop being incantations once you have to differentiate them. We learned that good visualisation is a debugging tool first and a teaching tool second: watching edges flicker between mint and pink as weights flip sign told us more about what the network was doing than any printed loss curve. We learned that streaming UIs are worth the complexity — a static post-hoc chart hides the dynamics that are the lesson. And we learned that small architectural decisions (a uniform model interface, a session-keyed in-memory store, NDJSON as the wire format) compound into a codebase that stays pleasant to work in well past the initial sprint. The platform is small, but every line of math is ours, and every gradient is one we can defend.
